# Use Cases

## Help
The user needs to understand how CourseForge works — what it does, how to navigate it, and how to get started.

## Brainstorming
The user has a lot of ideas and wants to capture them in a structured way so they can be referenced and used later in the course creation process.

## Check State
The user starts a clean session and needs to quickly understand what has already been done and what the next steps are.

## Setup
The user starts a new course by running a setup that creates the initial folder structure for the working environment.

## Coordinator
The user wants to be actively guided through the course creation process. The coordinator reads `progress.md` to understand the current state and drives the session forward by deciding what the next step is and executing it.

---

## Structure Phase

## Define Course
The user kicks off a new course by defining the three core constraints: audience, goal, and length.

## Define Scope
The user sets the boundaries of the course — what is explicitly included, what is out of scope, and what prerequisites are required.

## Define Modules
The user breaks the course into an ordered list of modules, each with a name and description. The user can view the full structure at any point and refine it — reordering, renaming, adding, or removing modules — until it feels right.

## Define Lesson Structure
The user decides the internal anatomy of a lesson — the sequence of sections every lesson will follow (e.g. review of previous lesson, theory, practice, test, conclusions). This becomes the standard template all lessons are built from.

---

## Design Phase

## Define Branding
The user establishes the visual identity for the course: colors, fonts, and any brand marks or logos.

## Create Templates
The user creates the reusable content and slide templates that all lessons will follow.

---

## Production Phase

## Write Lesson
The user writes the full content document for a specific lesson.

## Build Slides
The user turns a lesson's content document into a slide-by-slide presentation outline.

## Write Teacher Guide
The user creates the instructor-facing notes for a specific lesson.

---

## Review & Iteration

## Review Course
The user does a bird's-eye review of the full course to check for consistency, gaps, and alignment with the original goals.

## Refine Content
The user improves or rewrites an existing lesson or module.
